## 二、前端信息架构设计

### 核心问题

当前 StockReviewPanel 把所有卡片一股脑堆在一起，用户看不到重点、找不到信息。需要按**使用频率和重要性**分层：

### 设计原则：人话优先，关联优先

**当前问题**：前端显示的都是原始数据（如"主力净流入 -3200万"、"情绪周期：退潮"），用户看不懂这些数字代表什么，也看不到各卡片之间的关联。

**解决方案 — 两个核心原则**：

#### 原则一：人话翻译（每个数据必须有结论）

每个卡片不仅仅显示数字，还要显示**结论性描述**：

| 原始数据 | 人话翻译 |
|---------|---------|
| 主力净流入 -3200万 | "主力资金净流出3200万，近5日持续流出，资金在撤退" |
| 情绪周期：退潮 | "市场情绪明显退潮，今日涨停数较昨日减少40%，短线操作需谨慎" |
| 板块涨幅 1日+3.2% | "所属板块今日领涨市场，排名行业第2，板块效应强烈" |
| 换手率 25% | "换手率25%处于高位，远超该股近20日均值8%，说明筹码交换剧烈" |
| 5日均线位置 | "股价站上5日线，短期多头排列，但偏离度已达8%，存在回调压力" |

**实现方式**：
- 后端每个数据模块新增 `summary` 和 `conclusion` 字段，由规则引擎或LLM生成
- 折叠卡片的标题栏直接显示一句话摘要（如"市场环境：大盘偏强，个股相对跑赢"）
- 展开后先看人话解读，再看详细数据表格
- 数字旁边加趋势箭头（↑↓）和颜色（红涨绿跌），让方向一目了然

#### 原则二：关联串联（信息不是孤岛）

各卡片之间要建立关联，让用户看到整体图景：

**1. 头部"今日一句话总结"**
在股票核心信息区域，生成一段综合分析：
> "今日大盘偏强（上证+1.2%），市场情绪处于升温阶段。XX所属的半导体板块领涨（+3.5%），板块梯队完整、龙头明确。该股放量涨停，换手率25%处于高位，主力资金净流入5200万，量价配合良好。综合判断：板块效应+资金认可度高，建议关注次日开盘量能是否持续。"

**2. 卡片间交叉引用**
- 量价形态卡片中提到"所属板块今日领涨" → 可点击跳转到板块联动卡片
- 资金流向卡片中提到"与板块龙头资金流向一致" → 可点击跳转到龙头股对比
- 交易心理卡片中提到"类似3月15日的情况" → 可点击跳转到历史相似案例

**3. 决策复盘 × 深度分析打通**
- 6个因素面的"事件面"评分 → 关联到板块联动卡片中的驱动逻辑
- 6个因素面的"行业面"评分 → 关联到市场环境卡片中的板块排名
- 6个因素面的"风险面"评分 → 关联到情绪周期卡片中的风险信号

**4. AI解读增强**
AI解读不仅要解读个股本身，还要串联市场+情绪+板块的上下文：
- 市场大环境如何？对这只股票有什么影响？
- 情绪周期处于什么阶段？当前阶段下这类股票通常表现如何？
- 板块联动是否支持个股走势？龙头和中军的资金流向是否一致？

### 各深度分析卡片内容

每个卡片的结构统一为：

```
┌────────────────────────────────────────────────┐
│ [▶] 卡片标题（人话摘要一句话）                     │
│                                                │
│ 展开后：                                        │
│ ┌────────────────────────────────────────────┐ │
│ │ 📝 人话解读（2-3段）                          │ │
│ │   - 当前状态描述                              │ │
│ │   - 与个股的关联                              │ │
│ │   - 操作建议/风险提示                          │ │
│ ├────────────────────────────────────────────┤ │
│ │ 📊 详细数据                                   │ │
│ │   - 数字表格 + 趋势箭头 + 颜色                │ │
│ │   - 交叉引用链接（可跳转其他相关卡片）           │ │
│ └────────────────────────────────────────────┘ │
└────────────────────────────────────────────────┘
```

#### 市场环境卡片
- **人话摘要**：大盘强弱、赚钱效应、与个股的关系
- 四大指数涨跌幅（上证/深证/创业板/北证）
- 涨跌家数比 + 结论（如"赚钱效应良好，超7成个股上涨"）
- 涨停/跌停数 + 结论（如"涨停集中，封板质量高"）
- 市场风格标签（大盘/小盘/均衡）+ 说明
- 成交量/成交额Top5（精简显示）
- **关联**：链接到个股所属板块的市场表现

#### 情绪周期卡片
- **人话摘要**：当前所处阶段 + 该阶段的含义 + 对短线操作的建议
- 情绪周期阶段（大标签，颜色编码）+ 阶段详解
- 封板率、晋级率 + 解读（如"封板率65%，较昨日下降，说明资金犹豫"）
- 新闻讨论热度 + 解读
- 情绪周期原因 + 与个股走势的关联
- **关联**：链接到个股的历史相似案例

#### 板块联动卡片
- **人话摘要**：板块是否支持个股走势
- 所属板块名称及排名（1日/3日/10日）
- 板块内上涨比例 + 结论（普涨还是龙头带动）
- 龙头股代码 + 对比（涨幅/量能/资金流向）
- 梯队结构（龙头/中军/跟风）+ 完整性判断
- 驱动逻辑文本（LLM提取）
- **关联**：链接到资金流向卡片的板块排名

#### 资金流向卡片
- **人话摘要**：资金态度 + 是否支持当前走势
- 主力净流入（万元）+ 近5日趋势
- 超大单/大单/散户资金流向 + 各角色的态度解读
- 行业内排名 + 与龙头股对比
- **关联**：链接到量价形态卡片的放量/缩量判断

#### 量价形态卡片
- **人话摘要**：量价配合是否健康
- 5/10/20日均线位置 + 多头/空头排列判断
- 放量/缩量判断 + 与近20日均值的对比
- 换手率 + 分位值说明（如"处于近3个月的85分位"）
- 连板天数 + 历史最大连板对比
- **关联**：链接到板块联动卡片的板块表现

#### 交易心理卡片
- **人话摘要**：这次操作做对了什么/做错了什么
- 成功原因列表 + 可复制条件
- 失败原因列表 + 避免策略
- 心理分类标签（技术误判/消息遗漏/情绪驱动/计划不周/执行偏差）
- 可复制模式（具体可操作的条件）
- 预防策略（下次遇到类似情况怎么做）
- **关联**：链接到次日预案卡片

#### 次日预案卡片
- **人话摘要**：明天大概率面临什么情况，你该怎么准备
- 场景1（高开）：触发条件 + 操作策略 + 风险提示
- 场景2（平开）：触发条件 + 操作策略 + 风险提示
- 场景3（低开）：触发条件 + 操作策略 + 风险提示
- 关键价位（支撑位/压力位）+ 依据
- **关联**：链接到交易心理卡片的预防策略

#### 复盘历史卡片
- 假设性复盘记录列表 + 实时复盘按钮
- 策略选股记录列表 + 点击查看详情
- **关联**：相似案例对比

## 三、后端补全计划

### 3.1 资金流向模块（P1，1.5天）

**文件**: `src/stock_select/capital_flow.py`

**数据源**: 优先使用 AkShare `stock_individual_fund_flow` 接口，降级使用 daily_prices 推算（大单 ≈ 成交额前20%的交易）

**功能**:
- `build_capital_flow_report(conn, stock_code, trading_date)` → CapitalFlowReport dataclass
- 主力净流入、超大单、大单、散户流向
- 行业内排名

**数据库**: `capital_flow_daily` 表已存在

**集成**: 在 `_load_sector_quant` 中调用

### 3.2 自定义板块归类（P2，1天）

**文件**: `src/stock_select/stock_classifier.py`

**功能**:
- `classify_custom_sectors(conn, trading_date)` → 写入 stock_custom_sector 表
- 分类规则：
  - `limit_up_today`: 当日涨停
  - `high_turnover_today`: 当日换手率>20%
  - `high_turnover_10d`: 近10日平均换手率>15%
  - `unusual_10d`: 近10日内有3倍放量
  - `large_amount`: 当日成交额>10亿
- `get_custom_sector_tags(conn, stock_code, trading_date)` → 标签列表

**数据库**: `stock_custom_sector` 表已存在

### 3.3 Top10 展示（P1，0.5天）

**现状**: market_overview 已计算 top_volume/top_amount，但前端未展示

**改动**: 在 StockReviewPanel 的市场环境卡片中显示

### 3.4 融资融券（P3，1天，可选）

**数据源**: BaoStock `stk_margin` 接口

**风险**: BaoStock 在代理环境下不可用，需要标注为可选功能

2. 一眼看到决策复盘的6个因素面评分和解释
3. 点击展开任意深度分析卡片 → 看到对应的专业数据
4. 点击AI按钮 → 弹出AI解读摘要
5. 点击复盘历史 → 查看历史假设性复盘和策略选股记录
6. 右下角执行过程悬浮球 → 实时查看每一步的执行详情

所有豆包建议的核心复盘能力（A-G 中的 P1/P2 项）均已在前端可见、可操作。
